# 香港服务器推币机域名部署方案

## 推荐域名

将原示例里的 `demo.xhscopy.top` 替换为推币机项目域名：

```text
coinpusher.xhscopy.top
```

最终访问地址：

```text
https://coinpusher.xhscopy.top
```

## 当前部署状态

已部署到香港服务器：

```text
SSH Host: HK-Server
Tailscale IP: 100.74.21.72
Remote path: /var/www/coinpusher/dist
Nginx config: /etc/nginx/sites-available/coinpusher.xhscopy.top.conf
Nginx enabled: /etc/nginx/sites-enabled/coinpusher.xhscopy.top.conf
```

本机直接登录要使用 SSH config 里的别名：

```bash
ssh HK-Server
```

不要直接使用：

```bash
ssh root@100.74.21.72
```

原因：服务器接受的是 `~/.ssh/id_ed25519_xhs` 对应公钥；直接写 IP 时 SSH 默认尝试 `id_rsa`、`id_ed25519`，会出现：

```text
Permission denied (publickey)
```

如果必须直连，需要显式指定 key：

```bash
ssh -i ~/.ssh/id_ed25519_xhs -o IdentitiesOnly=yes root@100.74.21.72
```

## 部署结构

服务器目录建议：

```text
/var/www/coinpusher/
  dist/
    index.html
    assets/
```

本地项目已提供：

```text
deploy/deploy_hk.sh
deploy/nginx/coinpusher.xhscopy.top.conf
```

## 本地构建与上传

在本地项目目录执行：

```bash
./deploy/deploy_hk.sh HK-Server /var/www/coinpusher
```

脚本会执行：

1. `npm run build`
2. 通过 SSH 创建远程目录
3. 用 `rsync` 同步 `dist/` 到服务器

## Nginx 配置

将配置复制到服务器：

```bash
scp deploy/nginx/coinpusher.xhscopy.top.conf root@你的香港服务器IP:/etc/nginx/sites-available/coinpusher.xhscopy.top.conf
```

启用站点：

```bash
ssh HK-Server
ln -s /etc/nginx/sites-available/coinpusher.xhscopy.top.conf /etc/nginx/sites-enabled/coinpusher.xhscopy.top.conf
nginx -t
systemctl reload nginx
```

如果服务器使用的是 `/etc/nginx/conf.d/` 结构，则放到：

```text
/etc/nginx/conf.d/coinpusher.xhscopy.top.conf
```

## 阿里云 DNS 配置

如果 `xhscopy.top` 在阿里云管理：

| 配置项 | 值 |
| --- | --- |
| 记录类型 | A |
| 主机记录 | coinpusher |
| 记录值 | 香港服务器公网 IP，不是 Tailscale 100.x IP |
| TTL | 默认 10 分钟 |

注意：`100.74.21.72` 是 Tailscale 内网地址，只能在 tailnet 内访问。公网域名 `coinpusher.xhscopy.top` 应指向服务器公网 IP，除非你明确只想在 Tailscale 内部使用。

如果前面还有 CDN 或反向代理，也可以使用 CNAME：

| 配置项 | 值 |
| --- | --- |
| 记录类型 | CNAME |
| 主机记录 | coinpusher |
| 记录值 | CDN/代理提供的域名 |
| TTL | 默认 10 分钟 |

## HTTPS 证书

推荐使用 Certbot 自动签 Let's Encrypt：

```bash
apt update
apt install -y certbot python3-certbot-nginx
certbot --nginx -d coinpusher.xhscopy.top
```

Certbot 成功后会自动改写 Nginx 配置并启用 HTTPS。
