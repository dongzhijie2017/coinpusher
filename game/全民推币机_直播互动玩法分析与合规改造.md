# 全民推币机直播互动玩法分析与合规改造

## 结论

直播间互动版“全民推币机”的强点不是单纯推币，而是三层闭环：

1. 用户通过直播礼物进入玩法。
2. 推币过程提供物理反馈和持续消耗。
3. 九宫格触发“限时特权/购买权”，再驱动二次消费和主播打榜。

但这套机制不适合直接照搬到独立 Web 游戏或微信小游戏。尤其是“付费进入 + 概率触发 + 高倍率购买权 + 社交打榜价值”组合，很容易被平台审核或监管理解为变相博彩、付费抽奖或诱导消费。

我们的项目应保留“物理感应物 + 九宫格反馈 + 限时特权”的体验结构，但奖励必须改造成无现金、无提现、无实物、无交易、无主播打榜权益的游戏内成长奖励。

## 可借鉴机制

### 1. 物理感应物

直播版的“幸运鸭”本质上是一个移动 Sensor：

- 位于投币路径中段。
- 横向往复移动。
- 硬币碰撞后触发九宫格动画。
- 把纯粹掉落收益变成“投币过程中的中间惊喜”。

这很适合 Phaser + Matter.js：

```ts
this.duckSensor = this.matter.add.image(215, 220, "lucky-sensor", undefined, {
  isSensor: true,
  isStatic: true,
  label: "lucky-sensor"
});

this.tweens.add({
  targets: this.duckSensor,
  x: { from: 88, to: 342 },
  duration: 2800,
  yoyo: true,
  repeat: -1
});
```

实现注意：

- Sensor 只触发奖励动画，不施加物理推力。
- 同一枚硬币需要加冷却标记，避免在 Sensor 内多次触发。
- 九宫格结果应由前端固定表或服务端配置决定，正式版不要让客户端决定付费奖励。

### 2. 九宫格/转盘反馈

九宫格的作用不是“抽奖”文案，而是强化反馈节奏：

- 硬币掉落前已有一次中段触发。
- 玩家每次投币都多一个期待点。
- 动画可以制造节奏峰值。

合规表达建议：

- 用“机关触发”“图鉴扫描”“零件检索”。
- 不用“抽奖”“中奖”“爆率”“奖池”“开奖”。
- 九宫格可以展示固定道具、Buff、金币、任务进度。

### 3. 限时特权

“购买权”是直播版最强商业化点，但独立游戏里要降风险。

可以改成：

- 限时升级折扣。
- 限时皮肤折扣。
- 10 分钟金币加成。
- 自动收集道具。
- 离线收益加速。
- 主题图鉴碎片。

不建议做：

- 高倍率打榜权益。
- 与主播、公会、排行榜战力直接绑定的倍率礼物。
- 付费概率获得高价值购买资格。
- 任何可转让、可提现、可兑换实物的权益。

## 不建议照搬的高风险点

### 1. 送礼换游戏币

直播场景里“送礼”有平台语境，独立 Web/小游戏没有主播打赏语境。直接做成“送礼换游戏币”没有必要，也容易被理解为规避充值监管。

替代方案：

- 普通内购购买游戏内 Coin/Gem。
- 激励视频获得 Coin。
- 每日任务获得 Coin。
- 月卡/通行证给固定资源。

### 2. 概率获取高倍率购买权

直播版的“25 倍礼盒购买权”依赖主播打榜价值和限时损失厌恶。对我们项目来说，这一机制风险高于收益。

替代方案：

- 折扣券只影响游戏内皮肤、装饰、升级。
- 权益不可交易、不可赠送、不可提现。
- 概率触发的奖励不与真实货币价值强绑定。

### 3. 强限时二次付费

“24 小时购买权”容易形成强诱导消费。小游戏审核中，限时付费压力需要谨慎。

替代方案：

- 限时 Buff 直接发放，不需要二次付费。
- 折扣券有效期可以有，但用于低风险外观或升级。
- UI 文案避免“错过亏损”“最后机会”“只剩”等压迫式表达。

## 适合本项目的改造版规则

### 玩法流程

1. 玩家投币，硬币进入推币台。
2. 硬币经过中段移动 Sensor，例如“幸运机关”。
3. 触发九宫格“机关检索”动画。
4. 获得游戏内奖励或限时 Buff。
5. 硬币继续被推板推动，掉落后获得 Gold。
6. Gold 用于升级推板、投币速度、图鉴槽倍率。
7. Gem/皮肤/通行证作为商业化承接。

### 合规奖池示例

| 奖励 | 建议概率 | 说明 |
| --- | ---: | --- |
| 5 Gold | 18% | 小反馈 |
| 10 Gold | 10% | 中反馈 |
| 30 Gold | 4% | 小惊喜 |
| 1 Coin | 8% | 延长游戏时间 |
| 5 分钟 Gold +50% | 3% | 限时成长 Buff |
| 自动收集 30 秒 | 2% | 体验型道具 |
| 图鉴碎片 | 2% | 收集成长 |
| 升级 9 折券 | 1% | 消耗 Gold 的折扣 |
| 皮肤 8 折券 | 0.5% | 可选商业化承接 |
| 未触发额外奖励 | 51.5% | 控制产出 |

文案建议：

- “机关检索”
- “图鉴扫描”
- “发现零件”
- “获得 Buff”
- “解锁折扣”

避免：

- “抽奖”
- “中奖”
- “爆率”
- “奖池”
- “以小博大”
- “赢取”
- “返现”

## 对当前 Web MVP 的实现建议

### 第一阶段

- 增加一个移动 Sensor。
- 增加碰撞事件监听。
- 增加九宫格弹窗动画。
- 奖励只发 Gold/Coin/Buff，不接付费。

### 第二阶段

- 增加“机关检索”配置表。
- 增加 Buff 系统。
- 增加升级折扣券。
- 增加每日任务：触发机关 N 次。

### 第三阶段

- 接入广告奖励。
- 接入皮肤商店。
- 九宫格中的折扣券只用于皮肤和升级，不用于现金等价奖励。

## 商业化判断

直播版的核心利润来自“社交打榜价值 + 限时购买权 + 二次付费”，但这不适合我们的独立小游戏。

本项目更适合的商业化路径：

- 激励视频补 Coin。
- 去广告。
- 月卡。
- 通行证。
- 皮肤包。
- 固定礼包。
- 九宫格触发低风险 Buff 和折扣券。

这样能保留“投币过程有惊喜”的体验，同时不把玩法推向博彩或强诱导付费。
